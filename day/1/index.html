Print

To print something, we will use the methods below : 

> console.log("hello world")

> console.debug("hello world")
hello world

> console.error("hello world")
hello world

> console.info("hello world")
hello world
undefined
>

All prints exactly the same. Right ? And there are many more methods for logging or printing(actually to assist these or customize these functions output), but these are the 4 methods for printing. We will eventually find out when and where to use which one. For now we will be using console.log(...) only. Actually the don't print exactly the same, when we will be doing this on a browser or on a colorized environment, we will find out the difference.

And another question is wht there is a undefined at the end of every output ? - Just ignore it. And DON'T try to even search on Stackoverflow or Quora, and try to disble this. What actually happened here ? -

  • Line 1 > there was just two block, and variable was declared x with some value, from the inner block we tried to access that variable x and we have accessed it
  • Line 2 > Same as line 1, just made it a little clear
  • Line 3 > Declared an inner variable y
  • Line 4 > We just accessed an inner variable y, and it output the corret
  • Line 5 > We now accessed that same inner varible outside of that block, and we have successfully accessed that variable and that variable's value
  • Line 6 > Now insted of using var, we will use the keyword - 'let', and accessed that z, and everything is fine
  • Line 7 > We attempt to access that inner z from outside of z, but we get error 'z is not defiend', that's it. Cause as a block ends there is no more references of those variables.
